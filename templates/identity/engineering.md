# Engineering Style

<!-- Your coding style, architecture taste, workflow preferences. -->

## Code
- Clear names over comments
- Small functions, single responsibility
- Error handling: fail fast, wrap with context

## Git
- Conventional commits (feat:, fix:, refactor:, etc.)
- Small, focused PRs
- Rebase workflow

## Architecture
- Start simple, add abstraction only when pain demands
- Prefer composition over inheritance
- Explicit over implicit
